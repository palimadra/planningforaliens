Designing in the browser can be effortless

▼

You've tried designing in the browser but it kills your creativity.

You just can't get into the flow when you are coding.

You feel a little guilty. All the awesome people in our industry design in the browser.

And they have good reasons. If you did design in the browser you could:

  • Use Typekit right up front and focus on beautiful typography.
  • Make changes to colors and fonts easily and in one place.
  • Get feedback from clients on real, interactive prototypes.
  • Avoid creating PSDs for every conceivable breakpoint when you do a responsive design.
▼

Good news.

Designing in the browser can be as effortless as sketching.

And I'm putting together every tool and method you need to know into a book.

Sketching with CSS

You're a designer, not a developer, and this book keeps that in mind. But designing in the browser means writing code — that's the problem! The trick is to avoid getting bogged down in technical details.

With the tools and methods in this book you will:

  • Learn the techniques you need to know, and which you can safely ignore.
  • Get up and running quickly, without reading five more programming books.
  • Discover the newest CSS and HTML you should use when prototyping.
  • Finally understand how to use code in your design process.
